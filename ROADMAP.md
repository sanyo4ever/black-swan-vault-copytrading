# Roadmap

## Vision

Build the most transparent and accessible open-source copytrading infrastructure.

## Current focus (Q1-Q2 2026)

- Harden discovery quality filters and reduce false positives
- Improve monitoring efficiency for high-frequency traders
- Improve Telegram delivery reliability under rate limits
- Expand public catalog filtering/sorting UX
- Stabilize production observability and runbooks

## Next milestones

- Add pluggable source adapters (additional venues)
- Add ranking profile presets (conservative, balanced, aggressive)
- Add backtesting module for score calibration
- Add public contributor dashboard and benchmark datasets
- Add API docs for third-party builders

## Contributor-friendly backlog

- `good first issue`: docs, UI labels, validation, tests
- `help wanted`: connectors, scoring formulas, performance tuning
- `research`: data quality, anti-spam/anti-manipulation heuristics

## Out of scope for now

- Custodial user funds
- Automated user trading execution from this repo
- Promises of guaranteed returns

## How to propose roadmap changes

Open an issue titled `Roadmap proposal: <topic>` with:

- Problem statement
- Proposed solution
- Tradeoffs
- Implementation scope
